## 1. 项目成果

- 四杆机构运动学正解（Newton-Raphson + 解析解验证）
- 全范围扫描评估（传动角/死点/线性度 R²）
- 参数优化器 + 输出优先反求设计
- 机构动画可视化 + 特性曲线 + 原点/限位标注
- 完整 CSV 工程报告（6 区块）
- JSON 持久化 + 自动保存 + 撤消重做

## 2. 关键设计决策回顾

### 做对了的
1. **四层架构** — 改动核心算法时完全不影响 UI 和存储，隔离性好
2. **Newton-Raphson + 解析解双轨** — 互相验证，出 Bug 时快速定位
3. **200 点全范围扫描** — 比单点计算能发现更多问题（传动角低谷、线性度恶化区间）
4. **CSV 工程报告** — 6 区块格式化归档，实际工程中比 UI 数据更有用
5. **装配模式切换** — 同侧/对侧的显式选择避免了算法"悄悄选错解"
6. **写文档的纪律** — ADR + DevLog + DebugLog 让项目可追溯

### 可以更好的
1. **定位标注没有接入计算逻辑** — 画了限位线但不检查是否超出，需要手动判断
2. **输出优先搜索是同步的** — `ReverseDialog` 会卡 UI，应该放后台线程
3. **没有机构对比模式** — 想比较两组参数要手动截图
4. **预设库没有导出功能** — 无法把自己调好的参数一键分享

## 3. 经验教训

### 技术教训
1. IntelliSense 报红 ≠ 编译错误 — `.vscode/c_cpp_properties.json` 经常被忽略
2. 基距倾角是新 Bug 高发区 — Solver 改了但 View 没同步
3. 变量标签语义混乱（Min/Max 反了）— 只有使用者才能发现
4. `QDoubleSpinBox::valueChanged` + `QSlider::valueChanged` 会相互触发死循环 — 用 `blockSignals()`
5. `acos()` 参数必须夹逼到 [-1, 1] — 浮点误差时刻存在

### 工程教训
1. **不为了开发而开发** — 工具已经够用时，继续加功能是浪费时间
2. **CSV 归档的实用性远超预期** — 比在 UI 上看数据有用得多
3. **真实参数验证了工具的正确性** — report3 证明了从理论到实践的闭环
4. **教程的价值在于"下次用"** — 不是为了现在读完，是为了将来翻出来

## 4. 项目评估

### 作为"连杆计算器"
✅ 完成了设计目标 — 帮你找到了一套传动角 56°、R² 0.994 的参数
⚠️ 开发投入产出比偏低 — 几十小时开发 vs 半天手工计算

### 作为"学习训练项目"
✅ 极其高效 — 一个项目覆盖了 C++/Qt/CMake/机构学/数值方法/架构/测试/文档
✅ 产出 9 篇教程 + 1 篇开发手册 — 可复用的知识底座

### 最终定位
ServoLink 是一个**以航模连杆为载体的 C++ 桌面应用开发训练项目**。它首先是一本用真实代码写成的教科书，其次才是一个工程工具。
